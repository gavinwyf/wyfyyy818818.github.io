1. JavaScript的执行流程分为”同步”与”异步”

  1. JavaScript编程几乎总是伴随着异步操作，传统的异步操作会在操作完成之后，使用回调函数传回结果，而回调函数中则包含了后续的工作。

  2. 这也是造成异步编程困难的主要原因：我们一直习惯于“线性”地编写代码逻辑，但是大量异步操作所带来的回调函数，会把我们的算法分解地支离破碎。

  3. 此时我们不能用if来实现逻辑分支，也不能用while/for/do来实现循环，更不用提异步操作之间的组合、错误处理以及取消操作了。因此也诞生了如jQuery Deferred这样的辅助类库

  4. Deferred对象

传统的代码逻辑，通过嵌套回调函数，等待异步发送成功的通知后，在执行下一步操作，如果有大量的嵌套回调，耦合度，维护性，扩展性?

那么Deferred对象存在的意义在哪里?

对于开发者：

1：代码可读性，扁平化结构

2：让支离破碎的代码结构，继续保存线性的代码逻辑，也就是异步代码转为同步

3: 从抽象的角度，提供了一个抽象的非阻塞的解决方案（如 Ajax 请求的响应），它创建一个 “promise” 对象，其目的是在未来某个时间点返回一个响应。

deferred 可以理解为表示需要长时间才能完成的耗时操作的一种方式，相比于阻塞式函数它们是异步的，而不是阻塞应用程序等待其完成然后返回结果。deferred对象会立即返回，然后你可以把回调函数绑定到deferred对象上，它们会在异步处理完成后被调用。

所以，总的来讲Deferred通过一组 API 来规范化异步操作，这样也能够让异步操作的流程控制更加容易

3. 嵌套式回调

动画为例，下一个动画要等上一个执行完毕才可以继续，流程就会写到回调里面

//执行多个动画
$('ele1').animate({
    opacity: '.5'
}, 4000, function() {
    $('ele2').animate({
        width: '100px'
    }, 2000, function() {
        $('ele3').animate({
            height: '0'
        }, 2000);
    });
});

上面的代码编程逻辑也是正确的，但是针对这样的异步嵌套的回调逻辑，当我们的嵌套越多，代码结构层级会变得越来越深。

首先是阅读上会变得困难，其次是强耦合，接口变得不好扩展。我们需要一种模式来解决这种问题，这就是Promises所要做的事情。

4. Deferred(Promise)

为了让前端们从回调的地狱中回到天堂， jQuery 也引入了 Promise 的概念。 Promise 是一种令代码异步行为更加优雅的抽象，有了它，我们就可以像写同步代码一样去写异步代码。这个东东看起来很复杂，实际上我们只要抓住核心的使用就可以了

通过$.Deferred处理过的代码，很明显没有了回调的嵌套，虽然代码量看起来多了点，但是实际上，每一个代码执行部分都被封装了起来，只留了Deferred的接口处理了，等于是我们把执行的流程控制交给了Deferred，这样的好处就是我们在写嵌套函数的时候，可以用deferred提供的管道风格编写同步代码了

dtd.then(function() {
   //操作1
}).then(function() {
   //操作2
}).then(function() {
  //操作3
})
这里要了解3个步骤
var dtd = $.Deferred();  //创建
dtd.resolve();          //成功
dtd.then()              //执行回调

            width: "50%"
        }, 2000, function() {
            dtd.resolve(); // 改变Deferred对象的执行状态

    function animate2() {
        var dtd = $.Deferred(); // 生成Deferred对象
        $("#block4").animate({
            width: "50%"
        }, 2000, function() {
            dtd.resolve(); // 改变Deferred对象的执行状态
        });
        return dtd;
    }

    var anim = animate1();

    anim.then(function() {
        $("#block3").text('block3动画动画直接结束');

8. Promises 动画实例

如果同时执行多个动画的时候，我们就需要对每一个动画执行完毕后的处理下一个动作，意味着我们就要监听所有动画的执行完毕后的回调

使用promise和‘when’方式却可以很直截了当的表示： 一旦动画执行完成，就可以执行下一步任务。最终的结果是我们可以可以简单的用一个回调来解决多个动画执行结果的等待问题

when( function(){   
   //执行动画1
   return promise 
}, function(){  
  //执行动画2
  return promise 2 
} ).then(function(){  
   //当2个动画完成后，我们在自行后续的代码   
});

基本上可以用非阻塞的逻辑方式编写代码并异步执行。 而不是直接将回调传递给函数，这可能会导致紧耦合的接口，通过promise模式可以很容易区分同步和异步的概念。

9. 延迟对象相关API：

延迟对象，在jQuery的1.5引入，是通过调用jQuery.Deferred()方法创建一个可链式调用的工具对象。

它可以注册多个回调到回调队列， 调用回调队列，准备代替任何同步或异步函数的成功或失败状态。

延迟对象是可链式调用的，类似于jQuery对象是可链接的方式， 但它有其自己的方法。 创建一个Deferred（延迟）对象后，你可以通过使用下面任何方法 直接从对象创建或者链式调用 或 保存对象的一个变量，调用该变量的一个或多个方法。

deferred.always()

当Deferred（延迟）对象解决或拒绝时，调用添加处理程序。

deferred.done()

当Deferred（延迟）对象解决时，调用添加处理程序。

deferred.fail()

当Deferred（延迟）对象拒绝时，调用添加处理程序。

deferred.isRejected()

确定一个Deferred（延迟）对象是否已被拒绝。

deferred.isResolved()

确定一个Deferred（延迟）对象是否已被解决。

deferred.notify()

根据给定的 args参数 调用Deferred（延迟）对象上进行中的回调 （progressCallbacks）。

deferred.notifyWith()

根据给定的上下文（context）和args递延调用Deferred（延迟）对象上进行中的回调（progressCallbacks ）。

deferred.pipe()

实用的方法来过滤 and/or 链Deferreds。

deferred.progress()

当Deferred（延迟）对象生成进度通知时，调用添加处理程序。

deferred.promise()

返回Deferred(延迟)的Promise（承诺）对象。

deferred.reject()

拒绝Deferred（延迟）对象，并根据给定的args参数调用任何失败回调函数（failCallbacks）。

deferred.rejectWith()

拒绝Deferred（延迟）对象，并根据给定的 context和args参数调用任何失败回调函数（failCallbacks）。

deferred.resolve()

解决Deferred（延迟）对象，并根据给定的args参数调用任何完成回调函数（doneCallbacks）。

deferred.resolveWith()

解决Deferred（延迟）对象，并根据给定的 context和args参数调用任何完成回调函数（doneCallbacks）。

deferred.state()

确定一个Deferred（延迟）对象的当前状态。

deferred.then()

当Deferred（延迟）对象解决，拒绝或仍在进行中时，调用添加处理程序。

jQuery.Deferred()

一个构造函数，返回一个链式实用对象方法来注册多个回调，回调队列，调用回调队列，并转达任何同步或异步函数的成功或失败状态。

jQuery.when()

提供一种方法来执行一个或多个对象的回调函数， Deferred(延迟)对象通常表示异步事件。

.promise()

返回一个 Promise 对象用来观察当某种类型的所有行动绑定到集合，排队与否还是已经完成。

5. cookie: 保存key/value对儿的.txt文件

由服务器生成并存储在客户端文件系统(.txt格式)中的键值对
当浏览器再次请求该站点上的页面时，就会自动把保存的Cookie发回服务器

作用：Cookie使浏览器可在访问同一个站点的不同请求间传递数据，让服务器识别不同的客户端

有效期(生命周期)：

(1) expires值，定义生存周期，周期内Cookie有效，超出周期Cookie会被删除。

(2) 有些页面设为0或负值，关闭浏览器时，马上清除Cookie，不会记录用户信息，更安全。

(3) 若没有expires属性，创建的Cookie保存在内存中，浏览器一关闭就销毁。

作用域：默认情况下，站点保存的Cookie只能自己访问。但可以设置Cookie的domain和path值，限定哪个域名下的哪个路径可以访问Cookie

(1)何时创建： 只要在客户端持久保存个人数据，就用cookie

(2)何时使用： 向网站发送请求时，浏览器会自动携带该网站的cookie一并发送。

(3)如何创建cookie:  document.cookie="key=value;expires="+date;

   eg: document.cookie='uname=Tom';
   document.cookie='msg='+encodeURIComponent('Hi,JS你好');
   document.cookie='uid=7788;expires='+new Date().toGMTString();
   创建并保存三个Cookie

(4)读取cookie： document.cookie
            每个键值对以分号空格分隔
            *不包含expires属性*               

(5)如何读取每个属性和属性值
        先用split("; ")分隔每个键值对
        找到指定key的元素
        获取key对应的value

   eg: var cookies=document.cookie;
       var arr=cookies.split(";");   
       for(var i=0;i<arr.length;i++){
             var cookie = arr[i];
          var cookiePair = cookie.split("=");
          console.log(cookiePair[0]+'='+cookiePair[1]); //输出键值对
       }

  1. Bootstrap是一个HTML/CSS/JS框架，简化移动设备优先的响应式Web开发。

  2. Bootstrap的内容涉及到如下三个方面：

    (1) HTML： 为已有的H5标签新添了很多自定义属性 data-xxx

    (2)CSS： CSS Reset + 几百个class -> bootstrap.css

    (3)JS： 基于jQuery提供了几十个插件

  3. 知识点划分为五个部分：

    (1) 起步：      bootlint.js   html5shiv.js    respond.js

(2) 全局CSS样式：    容器、按钮、图片、表格、表单、栅格、排版、辅助类

(3) 组件：   字体图标、按钮组、下拉菜单、输入框组、导航、导航条、路径导航(面包屑)、分页、标签、徽章、巨幕、缩略图、警告框、进度条、媒体对象、列表组、面板、Well  

(4) jQ插件：  下拉菜单、按钮、模态框、标签页、工具提示、弹出框、警告框、Collapse(折叠框)、Carousel(轮播图)、Affix(附加侧边菜单)

(5) 定制：  LESS

(1) bootstrap.css  核心CSS文件，其中进行了CSSReset + 几百个class

(2) bootstrap.js   核心JS文件，基于jQuery提供了几十个插件

(3) html5shiv.js  由第三方提供，让老IE可以支持HTML5新标签

(4) respond.js  由第三方提供，让老IE可以支持CSS3新特性（媒体查询）

响应式布局

By Yunfeng

發表於 2015-08-27

响应式布局

一、响应式布局

Ethan Marcotte在2010年5月提出，简言之，是一个网站能够兼容多个终端—而不是为每个终端做一个特定版本。这个概念是为解决移动互联网浏览而诞生的。

二、响应式布局的优缺点

优点：

1.面对不同分辨率设备灵活性强
2.能够快捷解决多设备显示适应问题

缺点：

1.兼容各种设备工作量大，效率低下
2.代码累赘，会出现隐藏无用的元素，加载时间加长
3.其实这是一种折中性质的解决方案，多方面因素影响而达不到最佳效果
4.一定程度上改变了网站原有的布局结构，会出现用户混淆的情况

三、响应式布局怎么设计

1.CSS中的Media Query(媒体查询)

  • 通过不同的媒体类型和条件定义样式表规则。

  • 媒体查询让CSS可以更精确作用于不同的媒体类型和同一媒体的不同条件。

2.media query能获取到：

1）设备宽和高device-width，device-height 显示屏幕/触觉设备

2）渲染窗口的宽和高width，height 显示屏幕/触觉设备

3）设备手持方向，横向和竖向orientation(portrait|landscape)和打印机等。

4）画面比例aspect-ratio点阵打印机等

5）设备比例device-aspect-ratio-点阵打印机等

6）对象颜色或颜色列表color,color-index显示屏幕

7）设备的分辨率resolution

3.语法结构及用法

@media 设备名 only (选取条件) not (选取条件) and (设备选取条件)，设备二｛sRules｝

第一种： 在link中使用 @media

<link rel="stylesheet" type="text/css" media="only screen and (max-width:480px),
      only screen and (max-device-width:480px)" href="link.css" rel="external nofollow"/>
only可省略，限定于计算机显示器

第二种： 在样式表中内嵌 @media

@media (min-device-width:1024px)and(max-width:989px),       电脑显示器宽度>=1024(最大可见宽度为989)
        screen and (max-device-width:480px),                屏宽<=480手持设备
       (max-device-width:480px)and(orientation:landscape),  屏宽<=480的横向放置的手持设备
   (min-device-width:480px)and(max-device-width:1024px)and(orientation:portrait){srules}
                                                           屏宽>=480 <=1024 的垂直放置设备
 srules 为兼容设置的样式表   only(限定设备，可省略)

4.可用设备参数

all  handheld(手持设备) print(文件打印) screen(彩色电脑屏幕) speech braille(盲文) 
embossed(盲文打印)  tty(固定字母间距的网络媒体)  tv  projection(项目演示)

5.逻辑关键字

only  and  not  ,

6.可用设备名参数

width/height         输出设备中页面可见区域宽高(px)
device-width/height  输出设备的屏幕可见宽高(px)
orientation          设备手持方向：portrait横向|landscape竖向
aspect-radio         浏览器长宽比  width与height比率
device-aspect-radio  设备屏幕长宽比  device-width与device-height比率
color                每一组输出设备的彩色原件数，若不是彩色设备，则为0
color-index          输出设备的色彩查询表中的条目数，若没有采用彩色查询表，则为0
monochrome           一个单色框架缓冲区中每像素包含单色原件数，若非单色设备，则为0
resolution           定义设备的分辨率
scan                 定义电视类设备的扫描工序，progressive逐行扫描，interlace隔行扫描
grid                 查询输出设备是否使用栅格或点阵

7.为了更好的显示效果，需要格式化一些CSS属性初始值：

(1) 禁用iPhone中Safari的字号自动调整

html{
  -webkit-text-size-adjust:none;
}

(2) 设置HTML5元素为块

article,aside,details,figcaption,figure,footer,header,hgroup,menu,nav,section{
  display:block;
}

(3) 设置图片视频自适应调整

img{
     max-width:100%;
 height:auto;
 width:auto\9; //IE8
}
.video embed,.video object,.video iframe{
     width:100%;
 height:auto;
}

8.页面头部内需加入：

<meta  name="viewport" content="width=device-width;initial-scale=1.0">

meta viewport 属性是在移动设备上设置原始大小显示和是否缩放的声明

参数设置：

width/height       viewport的宽高
initial            初始的缩放比例
minimum-scale      允许用户缩放到的最小比例
maximum-scale      允许用户放大到的最大比例
use-scalable       用户是否可以手动缩放

9. 对于IE浏览器不支持media query，需要在页面头部引入css3-mediaqueries.js

<!--[if lt IE 9]>
   <script src="http ://css3-mediaqueries-js.googlecode.com/svn/trunk/css3-mediaqueries.js">
   </script>
<![endif]-->

另一种： (Bootstrap中 兼容性注释)

<!--[if lt IE 9]>
  <script src="js/html5shiv.min.js"></script>   IE支持HTML5
  <script src="js/respond.min.js"></script>     IE支持CSS3（尤其是媒体查询）
<![endif]-->

10. 如何编写响应式网页

(1) 声明viewport元标签

<meta name="viewport" content="width=device-width, user-scalable=no">

(2) 不使用绝对尺寸，使用相对尺寸

宽度：不使用px，使用%、auto
字体：不使用px，使用em、%

(3) 图片的宽度不使用px，使用%

img {
   /*width: 100%;*/
   max-width: 100%;   /*指定占父容器的百分比，且同时不能超过自身的原始大小——防止失真*/
}

(4) 使用div布局，浮动布局

(5) 有选择性的执行某些CSS样式——CSS3 Media Query技术

瀑布流布局

By Yunfeng

瀑布流布局

一、瀑布流布局：

它的特点是视觉表现的参差不齐的多栏布局，随着页面滚动条向下滚动不断加载数据到当前尾部。

最早采用瀑布流布局的是国外的pinterest网站，现在国内也相当很多。例如蘑菇街、美丽说。

二、实现方法：

JavaScript原生方法

JQuery 方法

CSS3的多栏布局

1.Javascript原生方法

采用 定位、浮动

图片外包裹两层盒子，大盒子处理盒子之间的布局关系，小盒子是图片的相框。

原理：

第一阶段：瀑布流的布局定位

1、计算列数=浏览器窗口宽度/图片宽度 图片定位使根据每一列数据库的高度计算接下来图片的位置

2、获取第一行图片的高度放到数组中，判断第二行第一张图片应出现的位置并做定位

3、第二行第一张图片要加在第一行最矮的图片下面
(第二行第一张的top值应该是第一行最矮的图片的高，left就是最矮图片的距最左边的距离）

4、获取最矮图片的索引值和高度值 getMinhIndex minH，设置第二行第一张位置

5、改变放第一行高度的数组，使最矮的图片高度改变为加上下面图片的高度和，使后面每一个图片都进行定位并改变高度。

第二阶段：瀑布流随滚动条动态添加

1、添加滚动条事件，监听滚动条

2、滚动条向下移动，页面向下移动，若最后一张图片加载一半时加载数据

3、获取最后一张图片加载一半时相对页面的高度，获取页面滚走高度和浏览器窗口可视区高度，判断是否加载图片

lastBoxH < scrollTop+height时，加载图片
//图片加载一半时页面高度
lastBoxH=oBoxs[oBoxs.length-1].offsetTop+Math.floor(oBoxs[oBoxs.length-1].offsetHeight/2);
//页面滚走的距离
scrollTop=document.body.scrollTop||document.documentElement.scrollTop;
//浏览器窗口可视区域的高度
height=document.body.clientHeight||document.documentElement.clientHeight;

4、自定义一个JSON对象，提供加载图片信息，动态添加图片到页面

for(var i=0;i<dataInt.data.length;i++){
          var oBox = document.createElement("div");
          oBox.className='box';
         oParent.appendChild(oBox);
         var oPic = document.createElement("div");
         oPic.className='pic';
         oBox.appendChild(oPic);
         var oImg = document.createElement('img');
         oImg.src="img/"+dataInt.data[i].src;
         oPic.appendChild(oImg);
}

2.JQuery方法

第一阶段：瀑布流的布局定位

原生JS方法中很多方法都可以省略掉，比如 获取对应类名的div方法，获取最矮图片索引方法。

具体代码如下：

$(window).on('load',function(){    waterfall();    })
function waterfall(){
  var $boxs=$('#main>div');
  var w=$boxs.eq(0).outerWidth();  //outerWidth()把padding、border都包含在内，width()则不会
  var cols=Math.floor($(window).width()/w);
  $('#main').width(w*cols).css('margin','0 auto');
  var hArr=[];
  $boxs.each(function(index,value){
      var h=$boxs.eq(index).outerHeight();
      if(index<cols){
          hArr[index]=h;
      }else{
          var minH=Math.min.apply(null,hArr);
          var minHIndex=$.inArray(minH,hArr);
          $(value).css({
            'position':'absolute',
            'top':minH+'px',
            'left':minHIndex*w+'px'
          })
          hArr[minHIndex]+=$boxs.eq(index).outerHeight();
      }
  })
}

第二阶段：瀑布流随滚动条动态添加

两大特点：链式操作，隐式迭代 (连续使用方法作用在同一个对象)

lastBoxDis = $lastBox.offset().top+Math.floor($lastBox.outerHeight()/2);
scrollTop = $(window).scrollTop();
documentH = $(window).height();

lastBoxDis < scrollTop+documentH时，动态添加图片

$.each(dataInt.data,function(key,value){       //dataInt是JS原生对象，不能直接使用value
      var oBox = $("<div>").addClass("box").appendTo($("#main"));
      var oPic = $("<div>").addClass("pic").appendTo($(oBox));
      $('<img>').attr('src',"img/"+$(value).attr('src')).appendTo($(oPic));
})

三、瀑布流三种方法比较

JS:

1、需要计算 计算列数=浏览器窗口宽度/图片宽度 图片定位使根据每一列数据库的高度计算接下来图片的位置

2、图片排序是按照图片计算的位置横向排列，位置是计算出来的，比较规范

CSS:

1、不需要计算，浏览器自动计算，只需要设置列宽，性能高

2、列宽随着浏览器窗口大小进行改变，用户体验不好

3、图片排序按照垂直顺序排列，打乱图片显示顺序

4、图片加载还是依靠Javascript实现
