瀑布流布局

一、瀑布流布局：

它的特点是视觉表现的参差不齐的多栏布局，随着页面滚动条向下滚动不断加载数据到当前尾部。

最早采用瀑布流布局的是国外的pinterest网站，现在国内也相当很多。例如蘑菇街、美丽说。

二、实现方法：

JavaScript原生方法

JQuery 方法

CSS3的多栏布局

1.Javascript原生方法

采用 定位、浮动

图片外包裹两层盒子，大盒子处理盒子之间的布局关系，小盒子是图片的相框。

原理：

第一阶段：瀑布流的布局定位

1、计算列数=浏览器窗口宽度/图片宽度 图片定位使根据每一列数据库的高度计算接下来图片的位置

2、获取第一行图片的高度放到数组中，判断第二行第一张图片应出现的位置并做定位

3、第二行第一张图片要加在第一行最矮的图片下面
(第二行第一张的top值应该是第一行最矮的图片的高，left就是最矮图片的距最左边的距离）

4、获取最矮图片的索引值和高度值 getMinhIndex minH，设置第二行第一张位置

5、改变放第一行高度的数组，使最矮的图片高度改变为加上下面图片的高度和，使后面每一个图片都进行定位并改变高度。

第二阶段：瀑布流随滚动条动态添加

1、添加滚动条事件，监听滚动条

2、滚动条向下移动，页面向下移动，若最后一张图片加载一半时加载数据

3、获取最后一张图片加载一半时相对页面的高度，获取页面滚走高度和浏览器窗口可视区高度，判断是否加载图片

lastBoxH < scrollTop+height时，加载图片
//图片加载一半时页面高度
lastBoxH=oBoxs[oBoxs.length-1].offsetTop+Math.floor(oBoxs[oBoxs.length-1].offsetHeight/2);
//页面滚走的距离
scrollTop=document.body.scrollTop||document.documentElement.scrollTop;
//浏览器窗口可视区域的高度
height=document.body.clientHeight||document.documentElement.clientHeight;

4、自定义一个JSON对象，提供加载图片信息，动态添加图片到页面

for(var i=0;i<dataInt.data.length;i++){
          var oBox = document.createElement("div");
          oBox.className='box';
         oParent.appendChild(oBox);
         var oPic = document.createElement("div");
         oPic.className='pic';
         oBox.appendChild(oPic);
         var oImg = document.createElement('img');
         oImg.src="img/"+dataInt.data[i].src;
         oPic.appendChild(oImg);
}

2.JQuery方法

第一阶段：瀑布流的布局定位

原生JS方法中很多方法都可以省略掉，比如 获取对应类名的div方法，获取最矮图片索引方法。

具体代码如下：

$(window).on('load',function(){    waterfall();    })
function waterfall(){
  var $boxs=$('#main>div');
  var w=$boxs.eq(0).outerWidth();  //outerWidth()把padding、border都包含在内，width()则不会
  var cols=Math.floor($(window).width()/w);
  $('#main').width(w*cols).css('margin','0 auto');
  var hArr=[];
  $boxs.each(function(index,value){
      var h=$boxs.eq(index).outerHeight();
      if(index<cols){
          hArr[index]=h;
      }else{
          var minH=Math.min.apply(null,hArr);
          var minHIndex=$.inArray(minH,hArr);
          $(value).css({
            'position':'absolute',
            'top':minH+'px',
            'left':minHIndex*w+'px'
          })
          hArr[minHIndex]+=$boxs.eq(index).outerHeight();
      }
  })
}

三、瀑布流三种方法比较

JS:

1、需要计算 计算列数=浏览器窗口宽度/图片宽度 图片定位使根据每一列数据库的高度计算接下来图片的位置

2、图片排序是按照图片计算的位置横向排列，位置是计算出来的，比较规范

CSS:

1、不需要计算，浏览器自动计算，只需要设置列宽，性能高

2、列宽随着浏览器窗口大小进行改变，用户体验不好

3、图片排序按照垂直顺序排列，打乱图片显示顺序

4、图片加载还是依靠Javascript实现

BOM基础知识整理

BOM里面我觉得最重要的还是计时器和事件处理这两个部分。简单整理一下，有许多平时不太常用。 我就整理一下我想复习的东西。

BOM：浏览器对象模型 专门访问浏览器窗口的对象模型

BOM提供了一些访问窗口对象的一些方法，我们可以用它来移动窗口位置，改变窗口大小，打开新窗口和关闭窗口，弹出对话框，进行导航以及获取客户的一些信息如：浏览器品牌版本，屏幕分辨率。

DHTML对象模型(7种)：

Window: 代表当前打开的浏览器窗口
history：当前窗口本次打开后的历史记录
navigator: 封装浏览器软件的版本和信息
document对象：当前浏览器窗口正在加载的网页文件（网页一加载，就会自动创建）
location: 封装了当前窗口正在打开的url地址
screen: 封装了当前窗口所在屏幕的信息
event：当事件发生时，用来封装事件的信息

  1. 打开/关闭窗口

    (1) 打开窗口：3种：

    1. 替换当前窗口：
      HTML： <a href="" target="_self"></a>
      BOM：  window.open("url","_self")
2. 在新窗口打开，可重复打开
      HTML：<a href="" target="_blank"></a>
      BOM: window.open("url")     //默认_blank
3. 在新窗口打开，不可重复打开
      HTML： <a href="" target="新窗口名"></a>
      BOM: window.open("url","窗口名")
*浏览器规定相同名称的窗口，只能打开一个*

    (2) 程序自动关闭窗口：window.close() 关闭当前窗口

  2. 对话框：

    1. 警告框：alert("提示信息");   ——只有确定按钮

2. 输入框：var str=prompt("提示信息");

3. 确认框：var bool=confirm("提示信息");——确定和取消  true   false

  3. 定时器：不用用户触发，也能自动执行的效果

    1. 周期性定时器setInterval：按固定时间间隔，反复执行相同操作

      step1. 使用一个全局变量保存定时器唯一编号   var timer=null;
       停止定时器，只能使用定时器的编号  

step2. 任务：一个函数：每次“循环”要执行的任务(任务结尾可设置结束条件)

step3. 何时启动、停止定时器：
        启动：timer=setInterval(任务函数 , 间隔毫秒数);
        停止：clearInterval(timer);  timer=null;

    2. 一次性定时器setTimeout：先等待一段时间，再执行一次任务，自动停止

step1. 全局  var timer=null;

step2. 任务：只执行一次     //好的习惯：任务结尾，timer=null;

step3. 何时启动/停止:
        启动：timer=setTimeout(任务 , 等待毫秒数);
        停止：clearTimeout(timer); timer=null;

1.window: 代表当前打开的浏览器窗口

window对象是BOM的根对象

window 对象属性：innerwidth,innerheight,pageXOffset,pageYOffset
       对象方法：alert(),confirm(),prompt() 三种对话框
                open(),close()，print()   blur(),focus() 失去、获取焦点
                setInterval(),setTimeout()  
                clearInterval(),clearTimeout()

2.navigator: 封装客户端浏览器和操作系统的信息

cookieEnabled: 判断当前浏览器是否启用了cookie
cookie：客户端存储个人信息的文件
userAgent: 客户端浏览器的名称和版本,判断浏览器的名称和版本
appName: 浏览器名称     
appVersion: 浏览器平台和版本信息

3. location: 封装了当前浏览器窗口正在打开的url

修改location.href属性，可替换当前窗口，打开新页面

属性：host,hostname,herf,pathname,port，protocol

方法：assign()加载新的  reload()重新加载  replace()替换当前文档

4. history: 当前窗口本次打开后的历史记录栈

前进：history.go(1) 等效于 history.forward()
刷新：history.go(0) 
后退：history.go(-1) 等效于 history.back()
     history的内容，禁止读写。
禁用后退：location.replace("新url");
      在history中，用新url，代替原url的位置。
      不会增加history中的历史记录数。

5. screen：封装了当前窗口所在屏幕的属性

width,height： 屏幕的实际分辨率
availWidth,availHeight:  去掉任务栏后的屏幕分辨率
window.innerHeight/Width:  浏览器窗口中文档显示区域的宽高

6. event对象：当事件发生时，自动创建的，封装事件属性的对象

(1) 获得event对象：

1.html: on事件名="事件处理函数(event)";       <div onclick="func(event);">div</div>  
    缺： 内容和脚本混合   优： 所有浏览器通用    

2.js:  var e=window.event||arguments[0];    
   IE: window.event 可直接使用event  无需传入参入
   FF: 使用event关键字将事件对象作为参数传入  function func(e){...}
   其他：arguments[0]

(2) event对象属性：

 1.获得目标对象： var src=e.srcElement(IE)||e.target(FF);   

         解决兼容性：function func(e){
              var obj = e.srcElement||e.target;
              alert(obj.nodeName);
             }

 2.事件发生时的鼠标位置：
    相对于浏览器显示区： e.x||e.clientX; e.y||e.clientY;
    相对于页面原点坐标： e.pageX;  e.pageY;
    相对于显示屏原点坐标：e.screenX; e.screenY;
    相对于事件源对象原点的坐标：e.offsetX; e.offsetY;

 3. keyCode/which/charCode : 键盘事件中按下的按键
    eventPhase :事件所处的传播阶段
    button : 鼠标哪个键被按下了
    cancelBubble : 是否取消事件冒泡
returnValue : 是否阻止事件默认行为

(3) 事件：

鼠标： onmouseover: 仅在跨入边界时触发一次
      onmousemove: 在范围内移动鼠标时，反复触发
      onmouseout: 仅在跨出边界时触发一次      
      onmouseup: 鼠标按下
      onmousedown: 鼠标抬起
      onclick: 鼠标点击后

键盘： onkeydown onkeyup onkeypress

状态： onload onunload
      onfocus onblur onchange          
      onsumbit onreset
      onresize onerror

(4) 定义事件：3种

1. 在元素开始标签中使用on事件名="事件处理函数"  <button onclick="calc()"></button> 

2. js中，对象.on事件名=事件处理函数对象  btn.onclick=calc;       btn.onclick=function(){}
   (一个事件定义只能绑定一个处理函数对象)

3. btnObj.attachEvent('onclick',function(){});   //IE
   btnObj.addEventListener('click',function(){});  //DOM

      处理兼容性：
          if(elem.attachEvent){
           IE:elem.attachEvent("on事件名",处理函数对象);
      }else{
          DOM: elem.addEventListener("事件名"，处理函数对象,是否在捕获阶段触发);
      }

(5) 事件周期：事件从发生到触发的全过程

IE事件周期：   2个阶段:目标触发-->冒泡
其它事件周期:   3个阶段：捕获-->目标触发-->冒泡  

        (1)捕获：最外层元素，向内，仅到目标元素，记录每个元素是否定义该事件     
              html 没有  body 没有                       
                    div d1 onclick="show()"
                div d2 onclick="show()"
            div d3 onclick="show()"
        (2)目标触发：触发d3的onclick，执行show方法
        (3)冒泡：由内向外，依次触发记录的事件
                执行div d2 onclick="show()"
                 执行div d1 onclick="show()";

(6) 取消冒泡：

e.cancelButtle=true;  //在事件处理函数末尾加，只执行一次，不冒泡
备用： e.stopPropegation();

利用冒泡：*如果多个子元素定义了相同的事件，只需要将相同的事件定义在父元素一次即可*

(7) 取消事件：2种

1.html： on事件名="return 处理函数()"
        function 处理函数(){...return true/false;}
          //*2个return必不可少 自己控制*      

2.js:  elem.on事件名=处理函数对象；  //处理函数的结尾，增加：e.returnValue=false;

          if(e.preventDefault){ 
            e.preventDefault(); 
          }else{
            e.returnValue=false; 
          }

DOM基础

DOM的知识非常重要，长时间用框架或库很容易忘掉DOM的经典但又复杂的操作，有时间还是回顾一下经典的DOM操作吧。我简单的整理了常用的基础DOM操作。不用就容易忘啊。

[增]：

创建元素添加元素：

Step1：创建节点：

var newNode=document.createElement("标签名"); //创建元素节点
var msg=document.createTextNode("text");  //创建文本节点
比如：var newa=document.createElement("a");  //<a></a>

Step2：设置节点的关键属性：

newNode.属性名="值"            //比如：newa.href="http ://tmooc.cn";
//newNode.setAttribute("属性名","值");                
newa.innerHTML="go to tmooc";  //<a href="http ://tmooc.cn">go to tmooc"</a>

Step3：将新节点，挂载到指定父元素下：2种：

1. 追加：parent.appendChild(newNode);
2. 之前插入：parent.insertBefore(newNode,oldNode);

文档片段：

1. var frag=document.createDocumentFragment(); //创建文档片段
2. frag.appendChild(新节点);
3. parent.appendChild(frag); //frag不会显示在页面中，用完立刻销毁

[删]：

删除元素：

1. var deleted=parent.removeChild(childNode); --删除
   或childNode.parentNode.removeChild(childNode);
2. var deleted=parent.replaceChild(newChild,oldChild); --替换

删除属性：

1. 核心DOM： elem.removeAttribute("属性名");
            var delnode = element.removeAttributeNode(attrNode); //返回被删除属性节点
2. HTML DOM：elem.属性名="";

[查]：

查元素：

1.var elem=document.getElementById("id值")；
2.var elems=node.getElementsByTagName("tagName");
3.var elems=node.getElementsByClassName("类名");   IE8，不支持（IE9+,FF3+,Safari3.1+,Chrome）
4.var elems=document.getElementsByName("name属性的值");只能用document找,专门于查找表单中的数据采集类元素

5.var elem=node.querySelector("任意选择器"); IE9+.. 没有兼容性问题,原生浏览器API，执行效率高??
6.var elems=node.querySelectorAll("任意选择器");   个别较新的选择器无法使用 eg. not
7.<body>-->document.body——常用

查节点：

1. parentNode  查找父节点
2. childNodes  查找所有子节点（含空格）
3. nodeName 节点名称 只读 （标签名，属性名，#text，#comment，#document）
4. nodeType 节点类型 元素1 属性2 文本3  只读
5. nodeValue 节点值 仅对文本节点有效 只读 （元素节点 undefined/null）

查属性：

1.遍历元素的属性： elem.attributes：返回属性数组
2.访问单一属性：  elem.attributes[下标]或["属性名"]
                 elem.getAttribute('属性名')
3.访问属性值：   elem.attributes[下标].value
                elem.getAttributeNode('属性名').value
4.判断元素是否设置了某个属性：elem.hasAttribute("属性名"); //true或false
                             elem.hasAttribute();  //是否拥有属性  IE8以下不支持

查CSS属性(可修改)：

(1)  var str=elem.style.css属性; //局限：仅能访问内联样式已定义的样式                 
(2)  var cssStyle=null;
     if(elem.currentStyle){   //IE  
     cssStyle=elem.currentStyle;
     }else{                  //其它浏览器
            cssStyle=getComputedStyle(elem,null);
     }                       //cssStyle中包含了所有最终应用到elem元素上的css属性
     var str=cssStyle.css属性;  //可以获取外部样式  CSS属性(驼峰命名)

DOM树中的节点关系：

父子关系： 

node.parentNode:访问node的父节点对象（不可能是文本,不受影响）                  
node.childNodes:访问node下的所有直接子节点
node.firstChild:访问node下的第一个子节点对象
node.lastChild: 访问node下最后一个子节点对象

兄弟关系：

node.previousSibling:访问node前一个相邻的兄弟节点
node.nextSibling:访问node后一个相邻的兄弟节点

节点树的问题：(除了parentNode都有空字符问题)

 第一个子    firstChild          firstElementChild

前一个兄弟   previousSibling      previousElementSibling
后一个兄弟   nextSibling          nextElementSibling

了解基于栈的CPU框架:

如果一个函数的局部变量定义在CPU的栈中，那么函数返回时它们的确就不存在了。

JS： 作用域链为一个对象列表，不是绑定的栈

  • 正常调用函数时，创建一个新对象保存局部变量，把这个对象添加至作用域链中。函数返回时，就从作用域链中将这个绑定变量的对象删除。

  • 若不存在函数嵌套，没有其他引用指向这个绑定对象，它就会当作垃圾回收掉。

  • 若存在函数嵌套，每个嵌套函数都各自对应一个作用域链，这个作用域链指向一个变量绑定对象。

  • 若这些嵌套的函数对象在外部函数中保存下来，则它们也会和所指向的变量绑定对象一样当作垃圾回收。

    若此函数：

    (1)定义了嵌套函数

    (2)并将它作为返回值返回或存储在某处的属性里

    这时会有一个外部引用指向这个嵌套函数。无法当作垃圾回收，且它所指向的变量绑定对象也不会被当作垃圾回收。

一、什么是CSS预处器

CSS预处理器定义了一种新的语言，基本思想是，用一种专门的编程语言，为CSS增加了一些编程的特性，将CSS作为目标生成文件，然后开发者就只要使用这种语言进行编码工作。

通俗的说，CSS预处理器用一种专门的编程语言，进行Web页面样式设计，然后再编译成正常的CSS文件，以供项目使用。CSS预处理器为CSS增加一些编程的特性，无需考虑浏览器的兼容性问题，例如你可以在CSS中使用变量、简单的逻辑程序、函数等等在编程语言中的一些基本特性，可以让你的CSS更加简洁、适应性更强、可读性更佳，更易于代码的维护等诸多好处。

CSS预处理器技术已经非常的成熟，而且也涌现出了很多种不同的CSS预处理器语言，比如说：Sass（SCSS）、LESS、Stylus、Turbine、Swithch CSS、CSS Cacheer、DT CSS等。如此之多的CSS预处理器，那么“我应该选择哪种CSS预处理器？”也相应成了最近网上的一大热门话题，在Linkedin、Twitter、CSS-Trick、知呼以及各大技术论坛上，很多人为此争论不休。相比过计我们对是否应该使用CSS预处理器的话题而言，这已经是很大的进步了。

到目前为止，在众多优秀的CSS预处理器语言中就属Sass、LESS和Stylus最优秀，讨论的也多，对比的也多。
本文将分别从他们产生的背景、安装、使用语法、异同等几个对比之处向你介绍这三款CSS预处理器语言。相信前端开发工程师会做出自己的选择——我要选择哪款CSS预处理器。

四、Sass、LESS和Stylus转译成CSS

Sass、LESS和Stylus源文件（除了LESS源文件在客户端下运行之外）都不能直接被浏览器直接识别，
这样一来，要正常的使用这些源文件，就需要将其源文件转译成浏览器可以识别的CSS样式文件，
这也是使用CSS预处理器很关键的一步，如果这一步不知道如何操作，那么意味着写出来的代码不可用。

三者如何将其源文件转译成所需要的CSS样式文件?

1.Sass源文件转译成CSS文件

“.sass”是Sass语言文件的扩展后缀名，而”.scss”是SCSS语言文件的扩展后缀名两者都是CSS预处理器语言，而且两者功能特性都一样，只是书写规则不同.

(1)创建一个Sass文件，可以命名为“style.scss”，并且将其放在对应的项目样式中，

如D盘的“www/workspace/Sass/css”目录下.

(2)启动Ruby的Command控制面板，找到需要转译的Sass文件，此例文件放置在D盘“www/workspace/Sass/css”

(3)如果仅针对于单文件的转译，需要将“style.scss”转译成“style.css”，只需要在对应的目录下输入：

$ sass style.scss style.css    
目录下增加了一个“style.css”文件，这个CSS文件可以用来调用，他就是“style.scss”转译出来的样式文件。

(4)我们不可能每保存一次这个Sass文件就来执行一回这个转译命令，可以通过下面的监听命令实现。

$ sass --watch style.scss:style.css

如果有多个Sass文件需要监听时，就很有必要的组织一下。默认情况之下，我喜欢把所有的Sass文件放在“/css/sass”目录中，而生成的CSS文件则直接放在“/css”目录下。

现在我们修改一下项目文件目录结构，在“/css”目录中再创建一个“sass”目录，并找刚才创建的“style.scss”文件移至“/css/sass”目录下。此时监听“css/sass”下的所有Sass文件时，可以在命令面板中输入：

$ sass --watch sassFileDirectory:cssFileDirectory    
冒号前面的“sassFileDirectory”是指“/css/sass”下的“sass”目录，
冒号后面的“cssFileDirectory”是指“css”目录，对应的命令就是：
$ sass --watch css/sass:css

除了使用Ruby的Command控制面板转译Sass之外还可以考虑第三方工具，比如说有名的Compass.app和fire.app。

2.LESS文件的转译成CSS文件

LESS文件和Sass文件转译可以说是大同小异，不同之处是LESS在安装的Node JS环境下通过其自己的命令来进行转译。

$ lessc style.less

上面的命令会将编译的CSS传递给stdout，你可以将它保存到一个文件中：

$ lessc style.less > style.css

除了上面的命令转译LESS源文件之外，现在还有很多第三方开发的工具，比较常见的有：SimpleLess、Less.app、LESS编译辅助脚本-LESS2CSS、WinLess和CodeKit.app等，我个人现在常用的是WinLess工具，简单易用，不过在IOS系统下LESS.app和CodeKit.app很好用。(我在慕客网LESS教程上使用的是Koala工具)

3.Stylus源文件转译成CSS文件

Stylus具有可执行性，因此Stylus能将自身转换成CSS。Stylus可以读取自“stdin”输出到“stdout”，
因此Stylus可以像下面转译源文件：

$ stylus –compress  <some.styl> some.css

Stylus也像Sass一样，同时接受单个文件和整个目录的转译。
例如，一个目录名为“css”将在同一个目录编译并输出“.css”文件。

$ stylus css

下面的命令将输出到“./public/stylesheets”：

$ stylus css –out public/stylesheets

还可以同时转译多个文件：

$ stylus one.styl two.styl

如果你的浏览器安装了Firebug，那么可以使用FireStylus扩展。

$ stylus –firebug  <path>

五、Sass、LESS和Stylus的语法

1.Sass语法

Sass3.0版本开始使用的是标准的CSS语法，和SCSS可以说是一样的。这样Sass代码转换成CSS代码变得更容易。

默认Sass使用“.scss”扩展名。Sass语法规则可以像CSS那样书写：

h1{
color:#936;
background-color:#333;
}

Sass也同时支持老的语法，老的语法和常规的CSS语法略有不同，他需要严格的语法，任何的缩进和字符的错误都会造成样式的编译错误。Sass可以省略大括号（｛｝）和分号（；），完全依靠严格的缩进和格式化代码，而且文件使用“.sass”扩展名，他的语法类似于：

h1
  color:#936
  background-color: #333

2.LESS语法

LESS是CSS的一种扩展形式，它并没有阉割CSS的功能，而是在现有的CSS语法上，添加了很多额外的功能。
就语法规则而言，LESS和Sass一样，都是使用CSS的标准语法，只是LESS的源文件的扩展名是“.less”.

h1{
color:#936;
background-color:#333;
}

3.Stylus语法

Stylus的语法花样多一些，它的文件扩展名是“.styl”，Stylus也接受标准的CSS语法，但是他也像Sass老的语法规则，使用缩进控制，同时Stylus也接受不带大括号（｛｝）和分号的语法.可以省略大括号（｛｝）

h1 
  color: #963;
  background-color: #333;

也可以省略大括号（｛｝）和分号（;）

h1
  color:#963
  background-color:#333

在Stylus样式中，你也可以在同一个样式文件中使用不同的语法规则，下面这样的写法也不会报错

h1 {
  color  #963
}

h2 
  font-size:1.2em

六、Sass、LESS和Stylus特性

这三款CSS预处理器语言具有一些相同的特性，例如：变量、混入、嵌套、函数等。

1.变量

a)Sass的变量 $

Sass声明变量必须是“$”开头，后面紧跟变量名和变量值，而且变量名和变量值需要使用冒号（：）分隔开。就像CSS属性设置一样：

/*声明变量*/
$mainColor: #963;
$siteWidth: 1024px;
$borderStyle: dotted;

/*调用变量*/                               /*转译出来的CSS*/

body {                                   |  body {
   color: $mainColor;                    |    color: #963;
   border:1px $borderStyle $mainColor;   |    border:1px dotted #963;
   max-width: $siteWidth;                |    max-width: 1024px;
}                                        |  }

b)LESS的变量 @

LESS样式中声明变量和调用变量和Sass一样，唯一的区别就是变量名前面使用的是“ @ ”字符

c)Stylus的变量

Stylus样式中声明变量没有任何限定，你可以使用“$”符号开始。结尾的分号（;）可有可无，但变量名和变量值之间的等号（=）是需要的。

注意：

(1)如果我们使用“ @ ”符号开头来声明（0.22.4）变量，Stylus会进行编译，但其对应的值并不会赋值给变量。

(2)在Stylus中不要使用“ @ ”符号开头声明变量。Stylus中调用变量的方法和LESS、Sass是完全相同的。

Stylus还有一个独特功能，不需要分配值给变量就可以定义引用属性：

    /*水平垂直居中*/                    /*转译出来的CSS*/
------------------------------------+------------------------------------
#logo                               |   #logo {
  position  absolute                |     position:absolute;
  top  50%                          |     top:50%;
  left  50%                         |     left:50%;
  width  w = 150px                  |     width:150px;
  height  h = 80px                  |     height:80px;
  margin-left  -(w / 2)             |     margin-left:-75px;
  margin-top  -(h / 2)              |     margin-top:-40px;
                    |   }

从上面的代码中我们可以看出，CSS预处理器语言中的变量是值级别的重复使用，可以将相同的值定义成变量统一管理起来.

CSS预处理器语言中变量的特性适用于定义主题（也就是我们常说的换肤），我们可以将背景颜色、字体颜色、边框属性等常规样式统一定义，这样不同的主题只需要定义不同的变量文件就可以。

2.作用域（Scope）

CSS预处理器语言中的变量和其他程序语言一样，可以实现值的复用，同样它也存在生命周期，也就是Scope（变量范围，开发人员习惯称之为作用域）简单说，就是局部变量和全局变量的概念

查找变量的顺序是先在局部定义中找，如果找不到，则查找上级定义，直至全局。

a)Sass的作用域

Sass中作用域在这三款预处理器是最差的，可以说在Sass中是不存在什么全局变量。

 $color: black;                           转义出来的样式：
.scoped {                            .scoped {
  $bg: blue;                            color:white; /*是白色*/
  $color: white;                         background-color:blue;
  color: $color;                     }
  background-color:$bg;              .unscoped {
}                                         color:white;/*白色（无全局变量概念）*/
.unscoped {                         }
  color:$color;
}

注意:

在Sass样式中定义变量，调用变量是没有全局变量一个概念存在，因此在Sass中定义了相同变量名时，在调用之时千万要多加小心，不然会给你的样式带来错误。

b)LESS的作用域

LESS中的作用域和其他程序语言中的作用域非常的相同，他首先会查找局部定义的变量，
如果没有找到，会像冒泡一样，一级一级往下查找，直到根为止，同样上面的例子，
我们来看看他在LESS下所起的变化。

@color: black;                               转译出来：
.scoped {                              .scoped {
  @bg: blue;                              color:white;/*白色（调用了局部变量）*/
  @color: white;                          background-color:blue;
  color: @color;                       }
  background-color: @bg;               .unscoped {
}                                           color:black;/*黑色（调用了全局变量）*/
.unscoped {                               }
  color: @color;
}

c)Stylus的作用域

Stylus虽然起步比较晚，但其作用域的特性和LESS一样，可以支持全局变量和局变量。会向上冒泡查找，直到根为止。

SASS与Compass（一）

By Yunfeng

發表於 2015-08-15

SASS与Compass基础

听一些大神都说，LESS和SASS很简单，根本不用看书，敲敲就明白了。我是不行啊，虽然东西不多，但是感觉不看点书，不踏实。

概述：

CSS开发是基于渲染原理和W3C标准来设计规则的，无算法，无数据结构，没有逻辑控制。

Sass在2007年，诞生于“工具控”的Ruby社区，开始服务的对象自然是Ruby工程师，是为了让他们能像写Ruby那样写CSS。

直到推出SCSS这种语法风格，Sass变得更像是CSS的扩展和增强，可以跟原生CSS混写。

Sass不能让你写出更好的CSS，但是却可以让你更高效的写出高可维护性的CSS。

QQ查找受益颇深的几点：

 1.Sass对原生CSS @import指令的扩展，让我可以在不同文件中的样式最后合并到一起。
2.变量机制，使得标准的颜色值、行高、列宽再也不用“东一榔头西一棒槌”的分散各处，有了一个统一的地方归纳整理。
3.标准样式的抽离，向调用函数一样，轻松将统一的样式抽离出来，避免了复制，一旦交互变动，只需修改抽离出来的混合器一处。
 4.样式继承的概念，使得我们对于查找中各种btn的修饰变得清晰，无论何种btn，统一从父级btn继承基础样式。
5.自动合图实在是太方便了，直接把切图扔到图片目录，对应的合图和CSS就全部自动生成了。

第一章、 Sass和Compass让样式表重焕青春

1. Sass支持两种语法。

(1)最初的缩进式语法 使用.sass为文件扩展名，对空格敏感，属性通过换行来分割。 

h1
  color:#000
  background:#fff

(2)SCSS即Sassy CSS在Sass3.0中被引入，它是CSS3的超集。SCSS文件使用.scss为扩展名，使用花括号和分号。

h1{color:#000;background:#fff}
注意：这两种语法可以在同一个项目中混合使用，只要不在同一个文件使用即可。

DRY(Don`t Repeat Yourself)

2.消除样式表的冗余

(1)通过变量来复用属性值

$company-blue: #1875e7;
h1#brand{color:$company-blue;}

(2)使用嵌套来快速写出多层级的选择器

ul.nav{float:right}
ul.nav li{float:left;}
ul.nav li a{color:#111}
ul.nav li.current{font-weight:bold;}

ul.nav{
  float:right;
  li{
    float:left;
    a{
      color:#111;
    }
    &.current{
     font-weight:bold;
    }
  }
}

(3)使用混合器来复用一段样式

 @mixin horizontal-list{
  li{
    float:left;
    margin-right:10px;
   }
 }
#header ul.nav{
  @include horizontal-list;
  float:right;
}
#footer ul.nav{
  @include horizontal-list;
  margin-top:1em;
}
使用 @mixin命令，可以将水平列表的规则抽取出来放到一个合适的混合器中。
然后调用 @include命令，将规则引入。因此，不再需要.horizontal-list类。

混合器可以和变量结合，能够根据参数来决定使用的样式。

@mixin horizontal-list($spacing:10px){
   li{float:left;}
}
#footer ul.nav{
  @include horizontal-list(20px);
  margin-top:1em;
}

(4) 使用选择器继承来避免重复属性

选择器继承的意思就是让一个选择器能够复用另一个选择器的所有样式，但又不重复输出这些样式属性。

.error{border:1px #f00; background:#fdd;}
.error .intrusion{font-size:1.2em; font-weight:bold;}
.badError{ @extend .error; border-width:3px;}

.badError继承父类.error，也就是复用父类的所有样式。

编译后：
.error,.badError{border:1px #f00; background:#fdd;}
.error .intrusion,.badError .intrusion{font-size:1.2em; font-weight:bold;}
.badError{border-width:3px;}

有时父类不需要在HTML中使用时，Sass3.2中引入占位选择器，它支持使用选择器继承的同时，不编译输出HTML中不使用的父类。

%button-reset{ ... }
.save{ @extend %button-reset;color:white;...}
.delete{ @extent %button-reset;color:white;...}

3.Compass

Compass是面向Sass的一个框架。

三个主要部分： 
      1）混合器
      2）实用工具的类库(能够集成到应用开发环境中的开发系统)
      3）一个用于构建框架和扩展的平台

使用Compass可以解决很多CSS问题
1）重置样式
2）不用计算创建布局
3）添加更专业样式
4）CSS3属性无需再写厂商前缀
