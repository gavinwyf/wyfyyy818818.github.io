fork同步相关问题：

Github现在越来越火了，这也说明了Github这个平台太有参与价值了，虽然我现在还只是个旁观者。
没事我会看看几个大神有没有更新什么新的作品或者小项目展示（嘿嘿，还可以偷偷的下载下来玩玩）。

Git方面教程有很多好的博客，比如 廖雪峰的Git教程非常好，本人很受益。

之前，刚用Github的时候，最头疼的就是同步fork，推送自己的文件或者分支合并到别人的上游目录中。刚开始我也是似懂非懂，有时候OK，有时候就报错。报错就开始翻译报的什么错，有时候真的就直接看报错提示就知道了，不要以为是一堆没用的English。其实，最简单的找错办法就是看报错提示了。看不懂查一下不就得了，又不费事，还锻炼英语水平。

废话不多说了，我们把我觉得有意思的部分整理一下，都是从网上前辈那里学的。

1. fork别人项目后如何与源项目保持同步：

(ps: 最直接简单暴力的办法，直接删除重新fork)

我们有时看到一些比较好的项目，我们都会fork一下，然后在本地操作，但是fork之后，项目是不会跟
源项目保持同步的，需要我们自己进行一些操作让其同步。

1） 打开git bash，进入要同步的仓库目录

2） 实现本地和远程同步，先将原项目pull到本地：

git remote add project http://github.com/源项目文件目录      //给源项目添加个别名
git pull project master      //pull project 下面的master分支

3) 将pull下来的更新 push到自己项目上面
git push

第一次push会报错，添加一个publickey就可以了。设置一下SSH就OK了，很简单。

SSH设置:

$ cd ~/.ssh/
$ ssh-keygen -t rsa -C "email@example.com"
或者
$ ssh-keygen 查看一下
$ cd .ssh
  修改id_rsa.pub(我用的是win8,linux直接vim修改就可以了 vim id_rsa.pub)
  修改内容就是你 Github 的 Deloy Keys里Add deloy key的内容
  重新git push就ok了

2.同步fork：

（1）给fork配置一个remote

1) git remote -v 查看远程状态

  eg.
   # origin  https://github.com/YOUR_USERNAME/YOUR_FORK.git (fetch)
   # origin  https://github.com/YOUR_USERNAME/YOUR_FORK.git (push)

‘

2) 添加一个将被同步给 fork 远程的上游仓库

  git remote add upstream https://github.com/ORIGINAL_OWNER/ORIGINAL_REPOSITORY.git

‘

3) 再次查看状态确认是否配置成功。

git remote -v
  # origin    https://github.com/YOUR_USERNAME/YOUR_FORK.git (fetch)
  # origin    https://github.com/YOUR_USERNAME/YOUR_FORK.git (push)
  # upstream  https://github.com/ORIGINAL_OWNER/ORIGINAL_REPOSITORY.git (fetch)
  # upstream  https://github.com/ORIGINAL_OWNER/ORIGINAL_REPOSITORY.git (push)

(2) 从上游仓库 fetch 分支和提交点，传送到本地，并会被存储在一个本地分支 upstream/master

git fetch upstream
  # remote: Counting objects: 75, done.
  # remote: Compressing objects: 100% (53/53), done.
  # remote: Total 62 (delta 27), reused 44 (delta 9)
  # Unpacking objects: 100% (62/62), done.
  # From https://github.com/ORIGINAL_OWNER/ORIGINAL_REPOSITORY
  #  * [new branch]      master     -> upstream/master

(3) 切换到本地主分支(如果不在的话)

git checkout master
 # Switched to branch 'master'

(4) 把 upstream/master 分支合并到本地 master 上，这样就完成了同步，并且不会丢掉本地修改的内容。

git merge upstream/master
 # Updating a422352..5fdff0f
 # Fast-forward
 #  README                    |    9 -------
 #  README.md                 |    7 ++++++
 #  2 files changed, 7 insertions(+), 9 deletions(-)
 #  delete mode 100644 README
 #  create mode 100644 README.md

(5) 如果想更新到 GitHub 的 fork 上，直接 git push origin master 就OK了。
