DOM基础

DOM的知识非常重要，长时间用框架或库很容易忘掉DOM的经典但又复杂的操作，有时间还是回顾一下经典的DOM操作吧。我简单的整理了常用的基础DOM操作。不用就容易忘啊。

[增]：

创建元素添加元素：

Step1：创建节点：

var newNode=document.createElement("标签名"); //创建元素节点
var msg=document.createTextNode("text");  //创建文本节点
比如：var newa=document.createElement("a");  //<a></a>

Step2：设置节点的关键属性：

newNode.属性名="值"            //比如：newa.href="http ://tmooc.cn";
//newNode.setAttribute("属性名","值");                
newa.innerHTML="go to tmooc";  //<a href="http ://tmooc.cn">go to tmooc"</a>

Step3：将新节点，挂载到指定父元素下：2种：

1. 追加：parent.appendChild(newNode);
2. 之前插入：parent.insertBefore(newNode,oldNode);

文档片段：

1. var frag=document.createDocumentFragment(); //创建文档片段
2. frag.appendChild(新节点);
3. parent.appendChild(frag); //frag不会显示在页面中，用完立刻销毁

[删]：

删除元素：

1. var deleted=parent.removeChild(childNode); --删除
   或childNode.parentNode.removeChild(childNode);
2. var deleted=parent.replaceChild(newChild,oldChild); --替换

删除属性：

1. 核心DOM： elem.removeAttribute("属性名");
            var delnode = element.removeAttributeNode(attrNode); //返回被删除属性节点
2. HTML DOM：elem.属性名="";

[查]：

查元素：

1.var elem=document.getElementById("id值")；
2.var elems=node.getElementsByTagName("tagName");
3.var elems=node.getElementsByClassName("类名");   IE8，不支持（IE9+,FF3+,Safari3.1+,Chrome）
4.var elems=document.getElementsByName("name属性的值");只能用document找,专门于查找表单中的数据采集类元素

5.var elem=node.querySelector("任意选择器"); IE9+.. 没有兼容性问题,原生浏览器API，执行效率高??
6.var elems=node.querySelectorAll("任意选择器");   个别较新的选择器无法使用 eg. not
7.<body>-->document.body——常用

查节点：

1. parentNode  查找父节点
2. childNodes  查找所有子节点（含空格）
3. nodeName 节点名称 只读 （标签名，属性名，#text，#comment，#document）
4. nodeType 节点类型 元素1 属性2 文本3  只读
5. nodeValue 节点值 仅对文本节点有效 只读 （元素节点 undefined/null）

查属性：

1.遍历元素的属性： elem.attributes：返回属性数组
2.访问单一属性：  elem.attributes[下标]或["属性名"]
                 elem.getAttribute('属性名')
3.访问属性值：   elem.attributes[下标].value
                elem.getAttributeNode('属性名').value
4.判断元素是否设置了某个属性：elem.hasAttribute("属性名"); //true或false
                             elem.hasAttribute();  //是否拥有属性  IE8以下不支持

查CSS属性(可修改)：

(1)  var str=elem.style.css属性; //局限：仅能访问内联样式已定义的样式                 
(2)  var cssStyle=null;
     if(elem.currentStyle){   //IE  
     cssStyle=elem.currentStyle;
     }else{                  //其它浏览器
            cssStyle=getComputedStyle(elem,null);
     }                       //cssStyle中包含了所有最终应用到elem元素上的css属性
     var str=cssStyle.css属性;  //可以获取外部样式  CSS属性(驼峰命名)

DOM树中的节点关系：

父子关系： 

node.parentNode:访问node的父节点对象（不可能是文本,不受影响）                  
node.childNodes:访问node下的所有直接子节点
node.firstChild:访问node下的第一个子节点对象
node.lastChild: 访问node下最后一个子节点对象

兄弟关系：

node.previousSibling:访问node前一个相邻的兄弟节点
node.nextSibling:访问node后一个相邻的兄弟节点

节点树的问题：(除了parentNode都有空字符问题)

               节点树                    元素树
      父     parentNode         parentElementNode
      子     childNodes         children
 第一个子    firstChild          firstElementChild
 最后一个    lastChild           lastElementChild
前一个兄弟   previousSibling      previousElementSibling
后一个兄弟   nextSibling          nextElementSibling

[改]：

修改元素：

1.elem.innerHTML: 读取或设置元素节点中HTML内容
   开始标签和结束标签之间的html文本    elem.innerHTML="";

2.elem.textContent: 开始标签和结束标签之间的纯文本内容

3.elem.innerText:  IE8支持(之前的不支持)，火狐不支持

    解决浏览器兼容性问题：
         if(elem.textContent){
            //使用elem.textContent
         }else{
            //使用elem.innerText
         }

修改属性：

1.核心DOM:

读取： var value=elem.getAttribute("属性名");
设置： elem.setAttribute("属性名","属性值");  IE8以下不支持
       elem.setAttributeNode(attrNode);

2.HTML DOM:

读取： var str=elem.属性名
设置： elem.属性名="值";
