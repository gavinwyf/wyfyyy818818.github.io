BOM基础知识整理

BOM里面我觉得最重要的还是计时器和事件处理这两个部分。简单整理一下，有许多平时不太常用。 我就整理一下我想复习的东西。

BOM：浏览器对象模型 专门访问浏览器窗口的对象模型

BOM提供了一些访问窗口对象的一些方法，我们可以用它来移动窗口位置，改变窗口大小，打开新窗口和关闭窗口，弹出对话框，进行导航以及获取客户的一些信息如：浏览器品牌版本，屏幕分辨率。

DHTML对象模型(7种)：

Window: 代表当前打开的浏览器窗口
history：当前窗口本次打开后的历史记录
navigator: 封装浏览器软件的版本和信息
document对象：当前浏览器窗口正在加载的网页文件（网页一加载，就会自动创建）
location: 封装了当前窗口正在打开的url地址
screen: 封装了当前窗口所在屏幕的信息
event：当事件发生时，用来封装事件的信息

  1. 打开/关闭窗口

    (1) 打开窗口：3种：

    1. 替换当前窗口：
      HTML： <a href="" target="_self"></a>
      BOM：  window.open("url","_self")
2. 在新窗口打开，可重复打开
      HTML：<a href="" target="_blank"></a>
      BOM: window.open("url")     //默认_blank
3. 在新窗口打开，不可重复打开
      HTML： <a href="" target="新窗口名"></a>
      BOM: window.open("url","窗口名")
*浏览器规定相同名称的窗口，只能打开一个*

    (2) 程序自动关闭窗口：window.close() 关闭当前窗口

  2. 对话框：

    1. 警告框：alert("提示信息");   ——只有确定按钮

2. 输入框：var str=prompt("提示信息");

3. 确认框：var bool=confirm("提示信息");——确定和取消  true   false

  3. 定时器：不用用户触发，也能自动执行的效果

    1. 周期性定时器setInterval：按固定时间间隔，反复执行相同操作

      step1. 使用一个全局变量保存定时器唯一编号   var timer=null;
       停止定时器，只能使用定时器的编号  

step2. 任务：一个函数：每次“循环”要执行的任务(任务结尾可设置结束条件)

step3. 何时启动、停止定时器：
        启动：timer=setInterval(任务函数 , 间隔毫秒数);
        停止：clearInterval(timer);  timer=null;

    2. 一次性定时器setTimeout：先等待一段时间，再执行一次任务，自动停止

step1. 全局  var timer=null;

step2. 任务：只执行一次     //好的习惯：任务结尾，timer=null;

step3. 何时启动/停止:
        启动：timer=setTimeout(任务 , 等待毫秒数);
        停止：clearTimeout(timer); timer=null;

1.window: 代表当前打开的浏览器窗口

window对象是BOM的根对象

window 对象属性：innerwidth,innerheight,pageXOffset,pageYOffset
       对象方法：alert(),confirm(),prompt() 三种对话框
                open(),close()，print()   blur(),focus() 失去、获取焦点
                setInterval(),setTimeout()  
                clearInterval(),clearTimeout()

2.navigator: 封装客户端浏览器和操作系统的信息

cookieEnabled: 判断当前浏览器是否启用了cookie
cookie：客户端存储个人信息的文件
userAgent: 客户端浏览器的名称和版本,判断浏览器的名称和版本
appName: 浏览器名称     
appVersion: 浏览器平台和版本信息

3. location: 封装了当前浏览器窗口正在打开的url

修改location.href属性，可替换当前窗口，打开新页面

属性：host,hostname,herf,pathname,port，protocol

方法：assign()加载新的  reload()重新加载  replace()替换当前文档

4. history: 当前窗口本次打开后的历史记录栈

前进：history.go(1) 等效于 history.forward()
刷新：history.go(0) 
后退：history.go(-1) 等效于 history.back()
     history的内容，禁止读写。
禁用后退：location.replace("新url");
      在history中，用新url，代替原url的位置。
      不会增加history中的历史记录数。

5. screen：封装了当前窗口所在屏幕的属性

width,height： 屏幕的实际分辨率
availWidth,availHeight:  去掉任务栏后的屏幕分辨率
window.innerHeight/Width:  浏览器窗口中文档显示区域的宽高

6. event对象：当事件发生时，自动创建的，封装事件属性的对象

(1) 获得event对象：

1.html: on事件名="事件处理函数(event)";       <div onclick="func(event);">div</div>  
    缺： 内容和脚本混合   优： 所有浏览器通用    

2.js:  var e=window.event||arguments[0];    
   IE: window.event 可直接使用event  无需传入参入
   FF: 使用event关键字将事件对象作为参数传入  function func(e){...}
   其他：arguments[0]

(2) event对象属性：

 1.获得目标对象： var src=e.srcElement(IE)||e.target(FF);   

         解决兼容性：function func(e){
              var obj = e.srcElement||e.target;
              alert(obj.nodeName);
             }

 2.事件发生时的鼠标位置：
    相对于浏览器显示区： e.x||e.clientX; e.y||e.clientY;
    相对于页面原点坐标： e.pageX;  e.pageY;
    相对于显示屏原点坐标：e.screenX; e.screenY;
    相对于事件源对象原点的坐标：e.offsetX; e.offsetY;

 3. keyCode/which/charCode : 键盘事件中按下的按键
    eventPhase :事件所处的传播阶段
    button : 鼠标哪个键被按下了
    cancelBubble : 是否取消事件冒泡
returnValue : 是否阻止事件默认行为

(3) 事件：

鼠标： onmouseover: 仅在跨入边界时触发一次
      onmousemove: 在范围内移动鼠标时，反复触发
      onmouseout: 仅在跨出边界时触发一次      
      onmouseup: 鼠标按下
      onmousedown: 鼠标抬起
      onclick: 鼠标点击后

键盘： onkeydown onkeyup onkeypress

状态： onload onunload
      onfocus onblur onchange          
      onsumbit onreset
      onresize onerror

(4) 定义事件：3种

1. 在元素开始标签中使用on事件名="事件处理函数"  <button onclick="calc()"></button> 

2. js中，对象.on事件名=事件处理函数对象  btn.onclick=calc;       btn.onclick=function(){}
   (一个事件定义只能绑定一个处理函数对象)

3. btnObj.attachEvent('onclick',function(){});   //IE
   btnObj.addEventListener('click',function(){});  //DOM

      处理兼容性：
          if(elem.attachEvent){
           IE:elem.attachEvent("on事件名",处理函数对象);
      }else{
          DOM: elem.addEventListener("事件名"，处理函数对象,是否在捕获阶段触发);
      }

(5) 事件周期：事件从发生到触发的全过程

IE事件周期：   2个阶段:目标触发-->冒泡
其它事件周期:   3个阶段：捕获-->目标触发-->冒泡  

        (1)捕获：最外层元素，向内，仅到目标元素，记录每个元素是否定义该事件     
              html 没有  body 没有                       
                    div d1 onclick="show()"
                div d2 onclick="show()"
            div d3 onclick="show()"
        (2)目标触发：触发d3的onclick，执行show方法
        (3)冒泡：由内向外，依次触发记录的事件
                执行div d2 onclick="show()"
                 执行div d1 onclick="show()";

(6) 取消冒泡：

e.cancelButtle=true;  //在事件处理函数末尾加，只执行一次，不冒泡
备用： e.stopPropegation();

利用冒泡：*如果多个子元素定义了相同的事件，只需要将相同的事件定义在父元素一次即可*

(7) 取消事件：2种

1.html： on事件名="return 处理函数()"
        function 处理函数(){...return true/false;}
          //*2个return必不可少 自己控制*      

2.js:  elem.on事件名=处理函数对象；  //处理函数的结尾，增加：e.returnValue=false;

          if(e.preventDefault){ 
            e.preventDefault(); 
          }else{
            e.returnValue=false; 
          }
