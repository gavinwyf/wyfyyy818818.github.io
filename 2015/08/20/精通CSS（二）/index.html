第八章 布局

所有CSS布局技术的根本都是3个概念：定位、浮动、外边框操纵

1.计划布局：

（1） 检查设计、寻找**重复模式**（包括结构中的模式或站点中反复出现的方式）

（2） 把页面划分为大的结构性区域 eg.容器、页眉、内容区域和页脚

（3） 内容区域本身，开始建立网格结构

（4） 在各内容区域中寻找不同的布局结构 （分为几列，开始详细设计结构和尺寸）

（5） 关注不同类型的内容 (是新闻稿、文章还是公告，每块内容起一个有意义的名字)

（6） 查看每个内容块的结构，看不同类型中是否有相同模式（应尽量采用一般的类名，然后根据上下文应用样式）

（7） 找出模式并确定命名约定后，定义将使用的元素

2.水平居中： 

margin： 0 auto;  

IE6及以下(混杂模式)：text-align:center; 误解为让所有东西居中，而不只是文本
这种方式使用text-align属性为一种hack，但是无害的。

3.浮动布局 （相对于绝对定位，浮动最容易使用，也是最可靠的）

* 两列浮动  （两列都向左浮动，或者一左一右浮动（可建立视觉隔离带）） 

     浮动元素中添加display：inline,防止IE中双边距浮动bug

* 三列浮动  （与双列布局类似，可在右列中再嵌入双列布局）

4.布局

* 固定宽度布局： px     (max-width min-width 控制宽度，避免失真或影响布局)

* 流失布局： 百分数

* 弹性布局： em（字号）
