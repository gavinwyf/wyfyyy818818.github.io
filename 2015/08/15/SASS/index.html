SASS与Compass基础

听一些大神都说，LESS和SASS很简单，根本不用看书，敲敲就明白了。我是不行啊，虽然东西不多，但是感觉不看点书，不踏实。

概述：

CSS开发是基于渲染原理和W3C标准来设计规则的，无算法，无数据结构，没有逻辑控制。

Sass在2007年，诞生于“工具控”的Ruby社区，开始服务的对象自然是Ruby工程师，是为了让他们能像写Ruby那样写CSS。

直到推出SCSS这种语法风格，Sass变得更像是CSS的扩展和增强，可以跟原生CSS混写。

Sass不能让你写出更好的CSS，但是却可以让你更高效的写出高可维护性的CSS。

QQ查找受益颇深的几点：

 1.Sass对原生CSS @import指令的扩展，让我可以在不同文件中的样式最后合并到一起。
2.变量机制，使得标准的颜色值、行高、列宽再也不用“东一榔头西一棒槌”的分散各处，有了一个统一的地方归纳整理。
3.标准样式的抽离，向调用函数一样，轻松将统一的样式抽离出来，避免了复制，一旦交互变动，只需修改抽离出来的混合器一处。
 4.样式继承的概念，使得我们对于查找中各种btn的修饰变得清晰，无论何种btn，统一从父级btn继承基础样式。
5.自动合图实在是太方便了，直接把切图扔到图片目录，对应的合图和CSS就全部自动生成了。

第一章、 Sass和Compass让样式表重焕青春

1. Sass支持两种语法。

(1)最初的缩进式语法 使用.sass为文件扩展名，对空格敏感，属性通过换行来分割。 

h1
  color:#000
  background:#fff

(2)SCSS即Sassy CSS在Sass3.0中被引入，它是CSS3的超集。SCSS文件使用.scss为扩展名，使用花括号和分号。

h1{color:#000;background:#fff}
注意：这两种语法可以在同一个项目中混合使用，只要不在同一个文件使用即可。

DRY(Don`t Repeat Yourself)

2.消除样式表的冗余

(1)通过变量来复用属性值

$company-blue: #1875e7;
h1#brand{color:$company-blue;}

(2)使用嵌套来快速写出多层级的选择器

ul.nav{float:right}
ul.nav li{float:left;}
ul.nav li a{color:#111}
ul.nav li.current{font-weight:bold;}

ul.nav{
  float:right;
  li{
    float:left;
    a{
      color:#111;
    }
    &.current{
     font-weight:bold;
    }
  }
}

(3)使用混合器来复用一段样式

 @mixin horizontal-list{
  li{
    float:left;
    margin-right:10px;
   }
 }
#header ul.nav{
  @include horizontal-list;
  float:right;
}
#footer ul.nav{
  @include horizontal-list;
  margin-top:1em;
}
使用 @mixin命令，可以将水平列表的规则抽取出来放到一个合适的混合器中。
然后调用 @include命令，将规则引入。因此，不再需要.horizontal-list类。

混合器可以和变量结合，能够根据参数来决定使用的样式。

@mixin horizontal-list($spacing:10px){
   li{float:left;}
}
#footer ul.nav{
  @include horizontal-list(20px);
  margin-top:1em;
}

(4) 使用选择器继承来避免重复属性

选择器继承的意思就是让一个选择器能够复用另一个选择器的所有样式，但又不重复输出这些样式属性。

.error{border:1px #f00; background:#fdd;}
.error .intrusion{font-size:1.2em; font-weight:bold;}
.badError{ @extend .error; border-width:3px;}

.badError继承父类.error，也就是复用父类的所有样式。

编译后：
.error,.badError{border:1px #f00; background:#fdd;}
.error .intrusion,.badError .intrusion{font-size:1.2em; font-weight:bold;}
.badError{border-width:3px;}

有时父类不需要在HTML中使用时，Sass3.2中引入占位选择器，它支持使用选择器继承的同时，不编译输出HTML中不使用的父类。

%button-reset{ ... }
.save{ @extend %button-reset;color:white;...}
.delete{ @extent %button-reset;color:white;...}

3.Compass

Compass是面向Sass的一个框架。

三个主要部分： 
      1）混合器
      2）实用工具的类库(能够集成到应用开发环境中的开发系统)
      3）一个用于构建框架和扩展的平台

使用Compass可以解决很多CSS问题
1）重置样式
2）不用计算创建布局
3）添加更专业样式
4）CSS3属性无需再写厂商前缀
